## 2026.4.27 Mon.
* 學到了indexOf，不明白為何跟typeof的寫法不一樣。
* 原來indexOf是Method，而typeof是operator，而length是property。
* typeof和 + 和 - 一樣是operator，就像一個外部監控員，不需要通過"."來呼叫內部功能
* String是一種Object，Object裡內建許多Property和Method，而"."表示「去尋找Object裡的Property or Method」
* 去找出Object裡的Property時，就像去找出一個人的身高體重一樣，只是去把一個資訊拿出來而已，所以通常不需要括號。
* 去找出Object裡的Method來執行，就像叫一個人去唱歌跑步一樣，需要執行某些操作或動作，因此通常需要括號來放入相關參數。
* 如果用variable.operator,例如name.typeof，系統會回傳undefined,表示查無此Property or Method.
* 這也可以解釋為何indexOf需要大寫O，而typeof不需要大寫O，因為indexOf是個變數，因此要符合camelCase.
